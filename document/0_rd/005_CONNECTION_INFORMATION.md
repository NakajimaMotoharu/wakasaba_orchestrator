# wakasaba_orchestrator 要件定義書：ConnectionInformation

## 基本情報

| 項目       | 内容                                                                        |
|----------|---------------------------------------------------------------------------|
| 完全修飾クラス名 | `com.wks.util.ConnectionInformation`                                      |
| ファイル名    | `ConnectionInformation.java`                                              |
| 種別       | record（public）                                                            |
| 責務       | SSH接続に必要な情報（ホスト・ポート・ユーザ名・秘密鍵パス）をイミュータブルに保持する。接続情報ファイルからのインスタンス生成メソッドを提供する |
| 主な依存クラス  | `WksConstants`                                                            |

---

## 要求機能

### CI-01：SSH接続情報のイミュータブルな保持

- SSH接続に必要な以下の4項目をイミュータブルに保持できること。
    - `host`：SSH接続先のIPアドレスまたはホスト名（`String`）
    - `port`：SSH接続ポート番号（`int`）
    - `user`：SSH接続ユーザ名（`String`）
    - `key`：SSH認証に使用する秘密鍵ファイルの絶対パス（`String`）
- Java の `record` 型を使用してイミュータビリティを保証すること。

### CI-02：接続情報ファイルからのインスタンス生成

- 指定されたファイルパスの接続情報ファイル（テキスト・UTF-8）を読み込み、`ConnectionInformation` のインスタンスを生成して返せること。
- ファイルの各行をそれぞれ host / port / user / key にマッピングすること（行番号は固定順）。
- `port` はファイル2行目の文字列を `int` へ変換すること。

#### 接続情報ファイルのフォーマット

| 行番号 | 項目   | 型   | 説明                  |
|-----|------|-----|---------------------|
| 1行目 | host | 文字列 | SSH接続先IPアドレスまたはホスト名 |
| 2行目 | port | 整数  | SSH接続ポート番号          |
| 3行目 | user | 文字列 | SSH接続ユーザ名           |
| 4行目 | key  | 文字列 | SSH認証用秘密鍵ファイルの絶対パス  |

### CI-03：ログ出力用文字列の生成（セキュア）

- `toString()` メソッドは `WksConstants.OTHER_SERVER_INFO` のフォーマットに従い、`host`・`port`・`user` のみを含む文字列を返すこと。
- `key`（秘密鍵パス）はログへ出力しないこと（セキュリティ要件）。

## 要求インタフェース

### レコードコンポーネント

| コンポーネント名 | 型        | 要件                     |
|----------|----------|------------------------|
| `host`   | `String` | SSH接続先のIPアドレスまたはホスト名   |
| `port`   | `int`    | SSH接続ポート番号             |
| `user`   | `String` | SSH接続ユーザ名              |
| `key`    | `String` | SSH認証に使用する秘密鍵ファイルの絶対パス |

### メソッド

| メソッド名           | 戻り値型                    | 修飾子             | 要件概要                                             |
|-----------------|-------------------------|-----------------|--------------------------------------------------|
| `getCiFromFile` | `ConnectionInformation` | `public static` | 指定ファイルパスを読み込み `ConnectionInformation` インスタンスを返す  |
| `toString`      | `String`                | `public`        | `key` を含まないセキュアなサーバ情報文字列を返す（recordデフォルトをオーバーライド） |

## 例外要件

| 例外クラス                            | 想定発生状況                          |
|----------------------------------|---------------------------------|
| `IOException`                    | ファイルが存在しない、読み取り権限がない等のファイル読込エラー |
| `NumberFormatException`（非検査）     | ファイルの2行目が整数値に変換できない場合           |
| `IndexOutOfBoundsException`（非検査） | ファイルの行数が4行未満の場合                 |

---

## 制約・注意事項

- Java の `record` 型のため、生成後にコンポーネントの値を変更できない。
- `Files.readAllLines()` はデフォルトでUTF-8を使用する。
- 接続情報ファイルは4行で構成されること。行末の空白は `Files.readAllLines()` によってそのまま保持されるため、ファイル作成時に注意すること。
- セキュリティ上、接続情報ファイルのパーミッションは `chmod 600`（所有者のみ読み書き可）に設定することを推奨する。
