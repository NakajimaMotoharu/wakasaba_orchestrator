# 補足資料：接続情報ファイル仕様

## 概要

本アプリケーションはコマンドライン引数として3つのサーバ接続情報ファイルのパスを受け取る。  
各ファイルは `ConnectionInformation.getCiFromFile()` によって読み込まれ、SSH接続情報としてパースされる。

---

## ファイルフォーマット

テキストファイル（UTF-8推奨）で、**1行1値**の形式。行の順番は固定。

```
<host>
<port>
<user>
<key>
```

### 各行の仕様

| 行番号 | 項目   | 型   | 説明                  | 例                        |
|-----|------|-----|---------------------|--------------------------|
| 1行目 | host | 文字列 | SSH接続先IPアドレスまたはホスト名 | `192.168.1.100`          |
| 2行目 | port | 整数  | SSH接続ポート番号          | `22`                     |
| 3行目 | user | 文字列 | SSH接続ユーザ名           | `mini`                   |
| 4行目 | key  | 文字列 | SSH認証用秘密鍵ファイルの絶対パス  | `/home/mini/.ssh/id_rsa` |

### ファイル例

```
192.168.1.100
22
mini
/home/mini/.ssh/id_rsa
```

---

## 実行コマンド例

```bash
java -jar wakasaba_orchestrator-1.0-SNAPSHOT-all.jar \
  /path/to/server0.txt \
  /path/to/server1.txt \
  /path/to/server2.txt
```

| 引数インデックス                 | 対象サーバ            | 処理内容                                       |
|--------------------------|------------------|--------------------------------------------|
| `args[0]` / `servers[0]` | サーバ0             | OSアップデート・アップグレード・再起動                       |
| `args[1]` / `servers[1]` | サーバ1（PaperMCサーバ） | PaperMC停止・OSアップデート・バックアップ・PaperMC更新・再起動・起動 |
| `args[2]` / `servers[2]` | サーバ2             | OSアップデート・アップグレード・再起動                       |

---

## 注意事項

- 行末の空白・改行コードに注意。`Files.readAllLines()` は改行文字を除いた行を返すが、行末スペースはそのまま含まれる。
- 2行目（port）が整数値に変換できない場合、`NumberFormatException`（非検査例外）が発生してプログラムが異常終了する。
- ファイルが4行未満の場合、`IndexOutOfBoundsException`（非検査例外）が発生する。
- 秘密鍵ファイルはSSH公開鍵認証に対応したOpenSSH形式を想定している（JSchの `addIdentity` に準拠）。
- セキュリティ上、接続情報ファイルのパーミッションは適切に制限すること（`chmod 600` 推奨）。
